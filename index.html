AI Ontslagbrief → Zorgtaken (Flexizorg)

Geautomatiseerde administratieve triage met verplichte menselijke goedkeuring (decision support).

1. INTAKE ONTSLAGBRIEF

Input

  • Secure upload (POC: één kanaal)
  • PDF / DOCX / scans
  • Virus-scan, metadata, versleutelde opslag

→ Output: Veilig opgeslagen document klaar voor extractie

2. EXTRACTIE & STRUCTURERING

  • Document Intelligence: layout parsing + OCR voor scans
  • Tekstextractie en structurering van kerninfo:
    • Diagnose/context (hoog niveau)
    • Zorgnoden
    • Medicatie/handelingen
    • Hulpmiddelen
    • Risico’s
    • Timing

→ Output: Gestructureerde zorginformatie

3. AI TAAKGENERATIE

  • AI pipeline (LLM + regels/taxonomie) met strikte JSON-output
  • Per taak:
    • Taakbeschrijving
    • Discipline/rol
    • Frequentie/duur/looptijd (indien vermeld)
    • Urgentie + startdatum
    • Benodigde competenties/attesten (waar mogelijk)
    • Confidence-score
    • Bronpassage (“evidence”)
  • Guardrails: geen taak zonder bronverwijzing

→ Output: Voorstel takenpakket met bronverwijzingen

4. HUMAN-IN-THE-LOOP REVIEW

  • Sociale dienst kijkt na, past aan, voegt toe, verwijdert
  • Audit trail: wie keurde wat goed, wanneer, en op basis van welke bronpassage

→ Output: Goedgekeurd takenpakket

5. PUBLICATIE NAAR FLEXIZORG

  • Mapping naar Flexizorg job/task model
  • Publicatie als draft/concept (POC: sandbox)
  • Terugkoppeling ID/status

→ Output: Takenpakket gepubliceerd op Flexizorg

6. MATCHING & INVULLING

  • Zorgprofessionals tekenen in
  • Time-to-fill en status terug naar sociale dienst

→ Output: Zorg effectief ingevuld na ontslag

7. MONITORING & LEARNING

  • Logging/monitoring, DLP, RBAC, retentie
  • KPI’s: tijdswinst triage, doorlooptijd discharge, acceptatiegraad voorstellen, fill-rate
  • Feedback loop: correcties terug naar model/regels

→ Output: Continu verbeterende AI-ondersteuning

PROGRAMMAFASERING

Fase 0 — POC

  • End-to-end flow: Upload → Extractie → AI taakvoorstel (evidence) → Review/approve → Draft post
  • Evaluatie op representatieve set ontslagbrieven
  • MVP scope-lock + Go/No-Go advies

Fase 1 — MVP (pilot-ready)

  • Inname via veilige upload of mailbox
  • Approval UI voor sociale dienst
  • Publicatie naar Flexizorg via API (concept → confirm → posten)

Fase 2 — Zorgstart-integratie (richting jouw voorstel)

  • Deze “AI ontslagbrief → taken” module wordt een kerncomponent binnen Zorgstart (ontslag = eerste businessinsteek)
  • Output wordt niet enkel een takenlijst, maar ook: probleemgebieden → zorgbundels → taken per rol → matchmaking → gedeeld zorgplan
  • Gezamenlijke cockpit: sociale dienst + Zorgstart/Nurseline opvolging (status, alerts, contextwijzigingen)

Kernprincipe: decision support met verplichte menselijke goedkeuring.
